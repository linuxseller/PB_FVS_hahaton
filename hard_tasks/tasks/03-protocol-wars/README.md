# 03. Protocol Wars


## Суть

Нужно написать сервис, который принимает куски сообщений, собирает их в правильном порядке, игнорирует дубликаты и повреждённые части.

Это автотестируемая версия сетевой задачи: вместо TCP-клиента ваш сервис получает пакеты через HTTP.

## Контракт решения

Контейнер слушает порт `8080`.

### Healthcheck

```http
GET /health
```

### Принять пакет

```http
POST /ingest
```

```json
{
  "message_id": "m1",
  "part": 2,
  "total": 3,
  "payload": "LLO ",
  "checksum": 283
}
```

Checksum считается просто:

```text
sum(ord(char) for char in payload)
```

Если checksum неверный, пакет надо игнорировать.

Ответ:

```json
{
  "accepted": true,
  "complete": false
}
```

### Получить результат

```http
GET /result/{message_id}
```

Если сообщение собрано:

```json
{
  "message_id": "m1",
  "status": "complete",
  "text": "HELLO WORLD"
}
```

Если нет:

```json
{
  "message_id": "m1",
  "status": "pending",
  "text": null
}
```

## Что будет в скрытых тестах

- части приходят не по порядку;
- дубликаты;
- повреждённые части;
- несколько сообщений одновременно;
- повторная отправка исправленной части;
- очень длинное сообщение.
