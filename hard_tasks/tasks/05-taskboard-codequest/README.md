# 05. TaskBoard CodeQuest


## Суть

Нужно реализовать сервис управления задачами с пользователями, правами доступа, дедлайнами и фильтрами.

Это автотестируемый аналог CodeQuest: скрытые тесты будут ломать крайние случаи.

## Контракт решения

Контейнер слушает порт `8080`.

### Healthcheck

```http
GET /health
```

## API

### Создать пользователя

```http
POST /users
```

```json
{
  "id": "u1",
  "name": "Anna",
  "role": "user"
}
```

Роли:

```text
user
admin
```

### Создать задачу

```http
POST /tasks
```

```json
{
  "id": "t1",
  "title": "Fix bug",
  "owner_id": "u1",
  "status": "todo",
  "deadline": "2026-05-01T12:00:00"
}
```

Статусы:

```text
todo
in_progress
done
```

### Получить задачу

```http
GET /tasks/{task_id}
```

### Обновить статус

```http
PATCH /tasks/{task_id}
```

```json
{
  "actor_id": "u1",
  "status": "done"
}
```

Пользователь может менять только свои задачи. Admin может менять любые.

### Удалить задачу

```http
DELETE /tasks/{task_id}
```

```json
{
  "actor_id": "u1"
}
```

Владелец может удалить свою задачу. Admin может удалить любую.

### Список задач

```http
GET /tasks?status=todo&owner_id=u1
```

Фильтры опциональны.

### Отчёт

```http
GET /report?now=2026-05-01T13:00:00
```

Ответ:

```json
{
  "total": 3,
  "by_status": {
    "todo": 1,
    "in_progress": 1,
    "done": 1
  },
  "overdue": 1
}
```

Просрочена задача, у которой:

```text
deadline < now и status != done
```

## Что будет в скрытых тестах

- неверные роли;
- удаление чужой задачи;
- неверные даты;
- фильтрация по нескольким параметрам;
- просрочка на границе времени;
- повторное создание одинакового id;
- несуществующий пользователь.
